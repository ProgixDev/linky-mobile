# Architecture Overview

## Philosophy

Code is a liability; the asset is the _system that produces correct code
cheaply_. This repo optimizes for: (1) agent legibility — everything an agent
needs is in-repo and greppable; (2) homogeneity — one obvious way to do each
thing, so pattern-matching works; (3) enforced constraints — taste lives in
lint rules and tests, not in review comments.

## Layers

```
src/app/        ROUTES (expo-router). Wiring only. No logic, no styling beyond layout glue.
src/features/   VERTICAL SLICES. ui/ + model/ + lib/ + __tests__/, public API via index.ts.
src/shared/     HORIZONTAL KIT. ui primitives, theme, lib (env, cn, id), testing utils.
```

Dependency direction is strictly `app → features → shared` (see
[module-boundaries](module-boundaries.md)). Features never know about each
other; composition happens at the route level.

## Data flow

```
user input ──▶ feature ui ──▶ zustand store (model/store.ts)
                                 │  actions validate via zod (model/schema.ts)
                                 ▼
                          AsyncStorage persistence (validated on rehydrate)
network (future) ──▶ api client in feature lib/ ──▶ zod-parsed DTOs ──▶ store
```

Rules: state lives in stores, not components (component state only for
ephemeral UI like a draft input); every edge is Zod-validated; derived data is
computed in selectors, never stored.

## Native layer

Continuous Native Generation (CNG): `ios/` and `android/` are build artifacts
generated by `expo prebuild` — never committed, never hand-edited. Native
configuration happens in `app.config.ts` via config plugins. Runtime version
policy is `fingerprint`, which lets EAS Update ship JS-only changes OTA and
forces a store build when native bits change.

## Reference implementation

`src/features/tasks/` exercises every pattern: schema-validated input,
persisted store with corrupt-data recovery, selector-based subscriptions,
NativeWind UI with shared primitives, Reanimated list animations, testIDs,
unit + component tests, and a Maestro CUJ flow. When in doubt, copy it.
