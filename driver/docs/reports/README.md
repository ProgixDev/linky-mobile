# Reports

Evidence reports live here as Markdown — the artifacts PMs and reviewers actually
read. This directory is intentionally excluded from `docs:lint` (links + orphans)
so generated reports never break the docs gate.

## What goes here

- **Feature reports** — `docs/reports/NNN-slug.md`, one per shipped feature: diff
  summary, Argent screenshots, acceptance-criteria-to-test traceability, and
  verification verdicts. Generated by the `/feature-report` skill at the end of
  feature work, before the PR.
- **Daily reports** — `docs/reports/daily/YYYY-MM-DD.md`, written in French and
  organized by project. Generated by the `/daily-report` skill so nothing about a
  day’s work gets forgotten.

## Conventions

- One report per file; never edit a shipped feature report in place — supersede it.
- Reference the spec (`specs/NNN-slug/`) or PRD the work delivered.
- Keep screenshots and recordings as relative links next to the report.
